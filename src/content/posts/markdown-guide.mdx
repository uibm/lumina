# Complete Markdown Formatting Guide

This guide demonstrates all the markdown formatting capabilities available in Lumina. Use it as a reference when writing your posts.

## Text Formatting

You can use **bold text** for emphasis, *italic text* for subtle emphasis, and ***bold italic*** for maximum emphasis. You can also use ~~strikethrough~~ for deleted content.

For inline code, wrap text in backticks: `const example = "inline code"`.

## Headings Hierarchy

# Heading Level 1
## Heading Level 2
### Heading Level 3
#### Heading Level 4
##### Heading Level 5
###### Heading Level 6

Use headings to create a clear content hierarchy. Each level is styled appropriately for visual distinction.

## Lists

### Unordered Lists

- First item in the list
- Second item in the list
- Third item with nested items:
  - Nested item 1
  - Nested item 2
    - Deeply nested item
- Fourth item back at root level

### Ordered Lists

1. First step in the process
2. Second step in the process
3. Third step with sub-steps:
   1. Sub-step A
   2. Sub-step B
   3. Sub-step C
4. Fourth step in the process

### Task Lists

- [x] Completed task
- [x] Another completed task
- [ ] Pending task
- [ ] Another pending task

## Links and References

You can create [inline links](https://astro.build) easily. Links are styled with accent colors and proper hover states.

You can also reference [the Astro documentation][astro-docs] using reference-style links for better readability.

[astro-docs]: https://docs.astro.build

## Blockquotes

> This is a blockquote. Use blockquotes for citations, important notes, or to highlight key information from external sources.
>
> Blockquotes can span multiple paragraphs. They're styled with a left border and subtle background to make them stand out from regular content.

Nested blockquotes are also supported:

> First level blockquote
>> Second level blockquote
>>> Third level blockquote

```typescript
interface BlogPost {
  title: string;
  description: string;
  publishDate: Date;
  tags: string[];
  featured: boolean;
}

async function getPost(slug: string): Promise<BlogPost | null> {
  const posts = await getCollection('posts');
```

### Python

```python
def calculate_reading_time(content):
    """Calculate estimated reading time in minutes."""
    words_per_minute = 265
    word_count = len(content.split())
    reading_time = round(word_count / words_per_minute)
    return max(1, reading_time)

post_content = "Your blog post content here..."
time = calculate_reading_time(post_content)
print(f"Estimated reading time: {time} minutes")
```

## HTML in Markdown

You can mix HTML when you need more control:

<div style="padding: 1rem; background: var(--color-surface); border-radius: var(--radius-md); border-left: 3px solid var(--color-accent);">
  <strong>Pro Tip:</strong> You can use HTML elements for advanced formatting when markdown isn't sufficient.
</div>
